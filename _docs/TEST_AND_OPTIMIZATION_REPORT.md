### 测试 3：EPS 推估算法
```javascript
// 推估公式验证
H1 实际 = Q1_2026 + Q2_2026  // 当有实际数据时
H2 推估 = H1 × (H2_2025 / H1_2025)  // 基于去年同比
年度 EPS = H1 + H2

示例：
Q1_2026: 15.50 元
Q2_2026: 16.20 元
H1 = 31.70 元

H1_2025: 28.00 元
H2_2025: 29.50 元
比例 = 29.50 / 28.00 = 1.054

H2_推估 = 31.70 × 1.054 = 33.41 元
年度 EPS = 31.70 + 33.41 = 65.11 元
```

## 🚨 已知限制与对策

### 限制 1：MOPS 网站安全限制
**问题**：直接爬蟲被拒绝
**对策**：
- ✅ 添加 User-Agent 和请求头
- ✅ 使用 CORS 代理备援
- ✅ FINANCIAL_DB 推估作为最后防线

### 限制 2：代理服务可靠性
**问题**：代理可能宕机或限流
**对策**：
- ✅ 多个代理备选
- ✅ 逐个重试
- ✅ 缓存策略（7天有效期）

### 限制 3：推估精度
**问题**：当无法爬蟲时，基于 2025 数据推估可能不准确
**对策**：
- ✅ 标记置信度为 "medium"
- ✅ 用户可见置信等级
- ✅ 计划集成 MOPS 爬蟲获取实际数据

### 限制 4：localStorage 容量
**问题**：2,327 家股票 × 4 季度数据可能超过 localStorage 限制
**对策**：
- ✅ 实现定期清理（7天过期）
- ✅ 监控 cache 大小
- ✅ 可选：改用 IndexedDB（未来优化）
