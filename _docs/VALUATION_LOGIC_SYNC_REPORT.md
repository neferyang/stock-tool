## 🎯 修復目標

解決系統三層估值邏輯的不一致問題，確保前端提示、表格評估、模型計算三者同步。

---

## 🔍 問題詳述

### 問題 1：P/E 判定不一致 ❌
**症狀**：
- 頁面頂部提示「P/E 已超過合理倍數」
- 但 P/E 本益比評估表格顯示 32.1x 為「合理」

**根因**：
- `calcVals()` 中使用的合理區間：`fairPE*0.8 ~ fairPE*1.3`
- `mkVerdict()` 中的 sell 判定硬編碼（未考慮產業 fairPE 差異）

**舉例**：半導體業 fairPE=25，合理上限應為 32.5x（25×1.3）
- 若當前 P/E=32.1x，應顯示「合理上限附近」而非「已超過合理倍數」

---

### 問題 2：低殖利率產業仍強制 GGM/股息模型 ❌
**症狀**：
- 台積電（半導體業，殖利率 0.92%）
  - 系統計算的 GGM 股息折現：560.27 元
  - 系統計算的殖利率防線：543.95 元
  - 現價：2365.00 元
  - **差距過大，造成投資者困惑**

**根因**：
- 系統對所有股票都計算 GGM、殖利率防線、殖利率合理價
- 但半導體業 fairYield=0%，表示該模型對此產業**不適用**
- 強行計算會得到虛假價格

**防呆邏輯**：
- 半導體業：fairYield=0（成長型產業）
- 計算結果無意義，應隱藏此模型並提示不適用

---

### 問題 3：FCF 標籤不清 ❌
**症狀**：
- 企業健康診斷卡顯示「自由現金流 (FCF)」
- 實際顯示的是百分比：`(FCF/netIncome * 100)%`
- 用戶不清楚這是絕對金額還是比率

**修正**：改為「FCF 利潤率」，清楚標示為百分比指標

---

```javascript
// 殖利率防線 - 成長型產業隱藏
if (div && div > 0 && !isGrowthIndustry) {
  // 計算並顯示
}

// GGM 股息折現 - 成長型產業隱藏並提示

  // 計算並顯示
} else if (isGrowthIndustry && div && div > 0) {
  // 顯示防呆提示：「該產業為成長型，股息折現模型不適用，建議使用 P/E 或 DCF 模型」
}

// 殖利率合理價 - 成長型產業隱藏
if (divYield && divYield > 0 && !isGrowthIndustry) {
  // 計算並顯示
}
```

**適用產業**（fairYield < 2%）：
- 半導體業（fairYield = 0%）
- 數位雲端（fairYield = 0%）

### 修復 2：P/E 判定同步（mkVerdict）

**實施位置**：`mkVerdict()` 函數中的 buy/sell 建議（Line 1444-1515）

#### Buy 建議（Line 1444+）
```javascript
// 基於產業 fairPE 的統一判定
const fairPE = params.fairPE || 16;
const lowerBound = Math.round(fairPE * 0.8);
const peRatioMsg = m.pe ? `P/E ${m.pe.toFixed(1)}x 低於 ${indName}合理下限 ${lowerBound}x` : '估值指標偏低';

// 建議文案改為動態生成
return {
  t: 'buy',
  ti: '🟢 估值偏低，可考慮買進或分批布局',
  bo: `【基本面】
• ${peRatioMsg}
• 多個估值模型顯示股價低於合理價值...`
}
```

#### Sell 建議（Line 1498+）
```javascript
// 基於產業 fairPE 的統一判定
const fairPE = params.fairPE || 16;
const upperBound = Math.round(fairPE * 1.3);
const peRatioMsg = m.pe ? `P/E ${m.pe.toFixed(1)}x 已超過 ${indName}合理上限 ${upperBound}x` : '估值指標偏高';

// 建議文案改為動態生成
return {
  t: 'sell',
  ti: '🔴 估值偏高，建議謹慎或考慮減碼',
  bo: `【估值分析】
• ${peRatioMsg}
• 股價溢價 15-20%...`
}
```

**優點**：
- ✅ 自動適應所有 33 種產業的 fairPE
- ✅ 避免硬編碼邏輯
- ✅ 前後文一致
