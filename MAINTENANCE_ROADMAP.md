# 🔧 維護與改進路線圖

根據當前 v6.2.0 系統狀態，以下是可進行的維護工作。

---

## 📊 **優先級分類**

### 🔴 **P0 - 立即執行（本週）**

#### **1. Phase 2.2：數據來源標註** ⭐ 推薦優先
**目標：** 為每個估值指標標記數據來源

**工作量：** 2-3 小時  
**難度：** ⭐⭐

**具體內容：**
```html
<!-- 改進前 -->
P/E: 15.2x

<!-- 改進後 -->
P/E: 15.2x (TWSE ✓)  <!-- 標記數據來源 -->
```

**實施步驟：**
1. 在 `go()` 函數中追蹤每個數據的來源
2. 修改 `calcVals()` 顯示來源標簽
3. 測試 2330、2207 驗證來源正確

**預期效果：**
- 用戶清楚知道每個指標來自哪裡
- 提高數據可信度
- 易於診斷數據不一致問題

---

#### **2. 虧損企業警告改進**
**目標：** 對無盈利企業提供更清晰的提示

**工作量：** 1-2 小時  
**難度：** ⭐

**具體內容：**
```javascript
// 改進前
EPS = N/A

// 改進後
❌ 無盈利企業
建議：使用 P/B 淨值法估值
參考產業：半導體 (fairPB: 6.0x)
```

**實施步驟：**
1. 在 `calcVals()` 中檢測 EPS ≤ 0
2. 提供替代估值方法
3. 建議使用的估值模型

---

### 🟡 **P1 - 本週內執行（下週初）**

#### **3. 數據優先級邏輯** 
**目標：** 自動選擇最可靠的數據源

**工作量：** 2 小時  
**難度：** ⭐⭐

**具體內容：**
```javascript
// 優先級選擇：TWSE > Yahoo > 計算值
if (m.pe_twse) {
  m.pe = m.pe_twse;
  m.peSource = 'TWSE';
} else if (m.pe_yahoo) {
  m.pe = m.pe_yahoo;
  m.peSource = 'Yahoo';
}
```

**預期效果：**
- 自動用最準確的數據
- 避免數據來源混亂

---

#### **4. 數據衝突警告**
**目標：** 當不同來源數據差異大時警告用戶

**工作量：** 1.5 小時  
**難度：** ⭐⭐

**具體內容：**
```
⚠️ 數據來源衝突
TWSE P/E: 15.2x vs Yahoo P/E: 16.5x (差異 7.9%)
建議：優先參考 TWSE 官方數據
```

---

#### **5. 特殊情況處理**
**目標：** 處理停牌、下市、新上市股票

**工作量：** 2-3 小時  
**難度：** ⭐⭐⭐

**具體內容：**

| 情況 | 處理方式 |
|------|--------|
| **停牌中** | 顯示「暫停交易」，使用最後已知價格 |
| **下市** | 顯示「已下市」，不提供估值 |
| **新上市** | 標記「新股」，歷史數據不足提示 |
| **退市風險** | 顯示警告標籤 |

---

### 🟠 **P2 - 下週執行**

#### **6. 參數微調與優化**
**目標：** 根據實測結果調整 fairPE、fairPB、dcfGrowth

**工作量：** 3-4 小時  
**難度：** ⭐⭐⭐⭐

**方法：**
1. 收集 10+ 股票的實際 P/E、P/B 數據
2. 與 fairPE、fairPB 比較
3. 調整參數至更貼近市場
4. 驗證估值準確度

**預期效果：**
- 估值結果更準確
- 推薦價位更有參考價值
