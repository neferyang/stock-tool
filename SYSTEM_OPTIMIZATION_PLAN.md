## 🔍 當前系統診斷

### 1. 性能問題

#### 問題 A：應用程式體積龐大
```
index.html : 3.1 MB
├─ 原因：HTML + CSS + JavaScript 全合一
├─ 影響：
│  ├─ 首次加載時間長（預期 3-5 秒）
│  ├─ 網路傳輸耗時多
│  └─ 內存佔用較大
```

**優化方案**：
- [ ] 模塊拆分（CSS / JavaScript 分離）
- [ ] 代碼壓縮（Minify）
- [ ] 資源優化（圖片、字體）
- [ ] 緩存策略（localStorage / IndexedDB）

---

#### 問題 B：API 請求失敗時的重試邏輯
```
當前流程：
1. TWSE API → 失敗
2. Yahoo Finance → 失敗
3. AllOrigins → 失敗
4. corsproxy → 失敗
5. 放棄（用戶看到「無法取得資料」）

耗時：約 40+ 秒（等待每個 API 超時）
```

**優化方案**：
- [ ] 設定更合理的超時時間（8 秒 → 5 秒）
- [ ] 並行請求（而非順序請求）
- [ ] 快速故障轉移（如果主要 API 失敗，立即使用本地備用）
- [ ] 緩存優先策略（如有快取就用快取，不用等 API）

---

#### 問題 C：數據計算效率
```
估值模型計算：
├─ P/E、P/B 計算
├─ DCF、GGM 計算
├─ 技術指標計算（RSI、MACD、K/D、Bollinger）
├─ 診斷卡評分
└─ 建議邏輯判定

預期總耗時：1-2 秒（若無 API 延遲）
```

**優化方案**：
- [ ] 使用 Web Worker（計算不阻塞 UI）
- [ ] 結果快取（同一股票連續查詢不重新計算）
- [ ] 算法優化（減少循環和冗餘計算）

---

### 2. 穩定性問題

#### 問題 1：API 失敗無備選
```
當前狀態：
❌ TWSE API 失敗 → 無資料顯示
❌ Yahoo Finance 失敗 → 無 K 線資料
❌ 所有代理都失敗 → 應用陷入等待

用戶體驗：頁面卡住，無法繼續
```

**優化方案**：
- [x] 實施本地備用數據（已暫存）
- [ ] 優雅降級（有什麼資料就用什麼）
- [ ] 清晰的故障提示（而非無限等待）

---

#### 問題 2：MOPS 爬蟲導致堆棧溢出
```
症狀：
❌ 應用偶爾崩潰
❌ 頁面卡住無回應
❌ 2026 年度 EPS 推估功能失效

根因：
爬蟲無遞迴限制 → 堆棧溢出 → 應用崩潰
```

**優化方案**：
- [x] 已禁用 MOPS 爬蟲（臨時方案）
- [ ] 完全移除爬蟲功能（永久方案）
- [ ] 或重寫爬蟲邏輯（添加遞迴限制）

---

#### 問題 3：錯誤處理不完善
```
當前問題：
❌ 未知錯誤時無提示
❌ 用戶不知道發生了什麼
❌ 無法自動復甦或手動重試

預期行為：
✅ 清晰的錯誤信息
✅ 重試按鈕
✅ 降級方案提示
```

**優化方案**：
- [ ] 統一錯誤處理（try-catch 全覆蓋）
- [ ] 用戶友好的錯誤信息
- [ ] 重試機制（手動或自動）
- [ ] 故障恢復指南

---

#### 問題 4：瀏覽器相容性
```
當前測試狀態：
⚠️ Chrome：主要開發環境
⚠️ Firefox：未充分測試
⚠️ Safari：未充分測試
⚠️ 移動端：未適配

預期：各瀏覽器都能正常使用
```

**優化方案**：
- [ ] 檢查 fetch() 相容性（IE 不支持）
- [ ] 檢查 Promise 相容性
- [ ] 檢查 localStorage 相容性
- [ ] 添加 Polyfill（必要時）
