## 🔍 詳細診斷

### 若右下角顯示「⚠️ 使用本地備援方案」

**可能原因和解決方案**：

```
1. 網路連接問題
   └─ 檢查：WiFi/VPN 是否正常
   └─ 解決：重新連接網路，重整頁面

2. TWSE/TPEX API 暫時故障
   └─ 檢查：https://openapi.twse.com.tw 是否可訪問
   └─ 解決：稍候 1-2 分鐘重試

3. 網路超時（連接慢）
   └─ 檢查：網路延遲是否過高
   └─ 解決：等待 10 秒後重整頁面

4. CORS 跨域問題（某些環境）
   └─ 檢查：瀏覽器 Console 有無 CORS 警告
   └─ 解決：使用代理或在服務器環境運行
```

**即使出現 ⚠️，應用仍可正常使用，只是不使用 L0.5 級，而是 L1-L4 級備援邏輯。**

```
   └─ 極罕見（API 格式不符預期）
   └─ 此時應檢查 Console 中的完整日誌
```

## 💬 常見問答

### Q1: 為什麼右下角顯示「⚠️ 使用本地備援方案」？
**A**: API 請求可能失敗或網路超時。但應用仍可用，會使用本地的 L1-L4 級備援邏輯，準確度略降至 95%。這完全正常。

### Q2: L0.5 和 L1 有什麼區別？
**A**:
- **L0.5**：官方 TWSE/TPEX API 直接映射，99.9% 準確
- **L1**：通過 API 返回的行業名稱進行正則匹配，95% 準確

L0.5 優先級更高，命中率更高，但如果 API 失敗則使用 L1 備援。

### Q3: 如果我看不到「[L0.5]」日誌是否表示失敗？
**A**: 不一定。可能是：
1. API 失敗（檢查 ⚠️ 狀態）
2. 該股票不在官方列表（新上市企業）
3. 代碼不符 TWSE/TPEX 格式

此時仍會使用 L1-L4 級判斷，結果仍可靠。

### Q4: 新上市企業何時能在 L0.5 中識別？
**A**: 企業上市後，TWSE/TPEX 官方會立即更新 API 數據。下一次應用啟動（或手動刷新）時會自動同步，無需代碼修改。

### Q5: 我應該多久驗證一次方案 B？
**A**: 
- **首次驗證**：立即（按上述步驟）
- **定期驗證**：建議每月檢查一次（確保 API 穩定）
- **無需主動維護**：系統完全自動化
