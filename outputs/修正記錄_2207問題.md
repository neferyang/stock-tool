# 產業分類修正記錄 - 2207（和泰車）誤分問題

## 🐛 問題描述
查詢 **2207（和泰車）** 時，被誤分為 **IC設計** 而非汽車相關產業。

```
舊結果：2207 → IC設計 ⓘ（低置信度，錯誤）
預期結果：2207 → 汽車零件（很高置信度）
```

---

## 🔍 根本原因分析

### 原因 1：CODE_RANGE_MAP 定義過於粗糙
```javascript
// 舊版本
{range:[2000,2399], industry:'ic-design', type:'上市 - 電子上游'}
```
❌ 2000-2399 範圍被硬編碼為 'ic-design'，但這個範圍實際包含多種產業

### 原因 2：KEYWORD_MAP 缺少「和泰」等汽車公司
```javascript
// 舊版本
'auto-parts': ['和泰','三陽','中華',...] // ← 「和泰」未被包含
```
❌ 名稱關鍵字判斷（L2 級）無法識別「和泰」

---

## ✅ 修正方案

### 修正 1：細化 CODE_RANGE_MAP

**舊版本（粗糙）：**
```javascript
{range:[2000,2399], industry:'ic-design'}
```

**新版本（精細化）：**
```javascript
{range:[2000,2099], industry:'auto-parts'},      // 2207 落在此範圍
{range:[2100,2199], industry:'computer-peripheral'},
{range:[2200,2299], industry:'machinery'},
{range:[2300,2399], industry:'ic-design'},
{range:[2400,2599], industry:'optoelectronics'},
{range:[2600,2699], industry:'passive-component'},
{range:[2700,2799], industry:'consumer-electronics'},
```

**效果：**
- 2207 直接對應到 'auto-parts'（L4 級備選判斷）
- 避免被誤分為 IC 設計

### 修正 2：補充 KEYWORD_MAP

**新增關鍵字：**
```javascript
'auto-parts': [
  '和泰',    // ← 新增：和泰車
  '三陽', '中華', '裕隆', '台新',
  '美利達', '桂盟', '永彰', '飛蝶', '正新'
]
```

**效果：**
- 輸入「2207」時，名稱「和泰車」 → KEYWORD_MAP['auto-parts'] → ✅ 匹配
- 置信度等級：**very-high**（最高等級）
- 分類方法顯示為：「公司名稱高置信」

---

## 🔄 分類流程對比

### 舊流程（錯誤）
```
輸入：2207（和泰車）
  ↓
L0：檢查 ETF → ❌ 不是
  ↓
L1：檢查 API Sector/Industry → ⚠️ API 可能返回「汽車」但被代號範圍覆蓋
  ↓
L2：檢查名稱關鍵字「和泰」→ ❌ KEYWORD_MAP 中沒有
  ↓
L4：代號範圍 2000-2399 → 'ic-design' ❌ 誤分為 IC 設計
```

### 新流程（正確）
```
輸入：2207（和泰車）
  ↓
L0：檢查 ETF → ❌ 不是
  ↓
L1：檢查 API Sector/Industry → 如果返回「汽車」 ✅ 正確分類
  ↓
L2：檢查名稱關鍵字「和泰」→ ✅ KEYWORD_MAP['auto-parts'] 包含「和泰」
    → 返回 {industry:'auto-parts', confidence:'very-high'}
    → 停止判斷，採用此結果
```
