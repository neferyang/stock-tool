## 🔍 驗證項目

### Phase 1：P/E 驗證
```
驗證邏輯：P/E = 現價 ÷ EPS
- [ ] 計算結果與 Yahoo Finance 對比
- [ ] fairPE 產業參數是否合理
- [ ] 極值處理（P/E > 500 是否過濾）
```

### Phase 2：P/B 驗證
```
驗證邏輯：P/B = 現價 ÷ BV（Book Value）
- [ ] BV 計算：BV = 現價 ÷ P/B
- [ ] fairPB 產業參數是否合理
- [ ] 負 P/B 數值是否正確處理
```

### Phase 3：ROE 驗證
```
驗證邏輯：ROE = 稅後淨利 ÷ 股東權益
- [ ] ROE 數值是否與官方財報一致
- [ ] ROE 單位是否正確（應為百分比）
- [ ] 負 ROE 是否標記異常
```

### Phase 4：買賣價位邏輯
```
驗證邏輯：
- 合理買進價 = EPS × fairPE
- 強力買進價 = EPS × fairPE × 0.75
- 合理賣出價 = EPS × fairPE × 1.2
- 停利參考價 = EPS × fairPE × 1.5

- [ ] 價位計算是否正確
- [ ] 價位排序是否合理（低→高）
- [ ] 股利防線計算：DIV ÷ fairYield
```

### Phase 5：異常數據處理
```
邊界情況：
- [ ] N/A 數據顯示（不計算估值）
- [ ] 零值處理（P/E=0, DIV=0）
- [ ] 極值處理（P/E>500, ROE>1000%）
- [ ] 負數處理（負 ROE、負 EPS）
- [ ] 缺失數據優雅降级（部分指標缺失）
```

### Phase 6：技術指標驗證
```
驗證邏輯：
- [ ] MA20、MA60、MA240 計算正確
- [ ] RSI 指標（14 日）
- [ ] MACD 計算（12、26、9 日）
- [ ] KD 隨機指標
- [ ] 布林通道（20 日，±2σ）
```

## 📊 驗證方法

### 方法 A：人工檢查（優先）
1. 打開應用 https://neferyang.github.io/stock-tool/
2. 逐個搜尋測試股票
3. 對比 Yahoo Finance 的官方數據
4. 檢查計算結果和建議邏輯

### 方法 B：計算驗證（備選）
手工計算示例：
```
AAPL（假設數據）：
- 現價：$310
- EPS：$8.27
- P/E = 310 ÷ 8.27 ≈ 37.48
- 合理 P/E = 22（科技股）
- 合理買進價 = 8.27 × 22 ≈ $181.94
- 強力買進價 = $181.94 × 0.75 ≈ $136.45
```
