## 🐛 修正問題

### 問題 1：當日按鈕未初始化
**現象**：頁面載入時，當日按鈕沒有被設為 active（藍色選中狀態）
- 按鈕存在於界面上
- 但沒有預設選中
- 用戶必須點擊才能看到當日走勢

**根本原因**：初始化代碼指向「6個月」按鈕而非「當日」按鈕
```javascript
// v4.11 的代碼
const dp=document.getElementById('dp');  // 這是 6個月 按鈕 ID
apb=dp;
dp.classList.add('active');
setPd(120,dp);
```

**修正方案**：改為查詢當日按鈕並初始化
```javascript
// v4.12 的代碼
const dailyBtn = document.querySelector('button[onclick="setPd(-1,this)"]');
if(dailyBtn) {
  apb=dailyBtn;
  dailyBtn.classList.add('active');
  setPd(-1,dailyBtn);
}
```

**結果**：✅ 頁面載入時當日按鈕預設為藍色 active 狀態

### 問題 2：本益比說明區間計算錯誤
**現象**：用戶截圖中顯示「半導體業合理區間：18-25x」
- 但當前 P/E=32.6x，被判定為「偏高」
- 合理區間計算邏輯不清晰

**根本原因**：本益比說明使用了 `fairPE*0.7 ~ fairPE`
- 半導體業 fairPE=25x
- 計算結果：17.5 ~ 25x ≈ 18-25x
- 但這與評估邏輯不符（評估以 0.8 和 1.3 倍為界）

**修正方案**：使用正確的評估區間與顯示 fairPE 值
```javascript
// v4.11 的說明
`目前P/E=${f(pe,1)}（${indName}合理區間：${Math.round(fairPE*0.7)}~${fairPE}x）`
// 結果：目前P/E=32.6（半導體業合理區間：18-25x）

// v4.12 的說明
`目前P/E=${f(pe,1)}（${indName}合理區間：${Math.round(fairPE*0.8)}~${Math.round(fairPE*1.3)}x，fairPE=${fairPE}x）`
// 結果：目前P/E=32.6（半導體業合理區間：20-33x，fairPE=25x）
```

**評估邏輯說明**：
```
P/E < fairPE*0.8（20x）    → 低估 ✓
fairPE*0.8～fairPE*1.3     → 合理（20-33x）
P/E ≥ fairPE*1.3（33x）    → 高估 !
```

**結果**：✅ 用戶現在可以清晰看到：
- 當前 P/E：32.6x
- 合理區間：20-33x
- fairPE 基準：25x
- 評估：偏高（因為 32.6 接近但未超過 33 的上限）
