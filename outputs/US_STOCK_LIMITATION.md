# 美股數據限制說明

**發現日期**：2026-06-04  
**版本**：v4.13  
**狀態**：已識別，文檔記錄

---

## 🔍 美股查詢的限制

### 現象描述

查詢美股（如 AAPL - Apple Inc）時出現：

1. **基本面數據全部 N/A**
   - 本益比 P/E：N/A ❌
   - 股價淨值比 P/B：N/A ❌
   - EPS：N/A ❌
   - 現金股利：N/A ❌
   - 股利殖利率：N/A ❌

2. **產業分類失敗**
   - 產業：未分類（low）❌
   - 應該分類為：應該是科技類或電腦設備業

3. **估值模型無法計算**
   - 「財務資料不足，無法計算」❌

---

## 🔧 技術根本原因

### Yahoo Finance API 限制

**美股數據可用性**：
```
✓ 股票代碼、公司名稱、當前價格
✓ 52週高低點
✓ K線歷史數據（日線）
✓ Sector / Industry（不穩定）

✗ P/E 比（通常返回 null）
✗ EPS（通常返回 null）
✗ P/B 比（通常返回 null）
✗ 現金股利（通常返回 null）
✗ ROE、PEG 等衍生指標
```

**原因分析**：
- Yahoo Finance 免費 API 對美股財務數據有限制
- 返回的 quoteSummary 模塊中 financialData 字段常為空
- defaultKeyStatistics 也經常缺少關鍵數據

**API 端點**：
```
GET /v10/finance/quoteSummary/{symbol}?modules=
  - price (✓有數據)
  - summaryDetail (△不穩定)
  - defaultKeyStatistics (△不穩定)
  - financialData (✗幾乎無數據)
```

| 數據項 | 台股 | 美股 | 上櫃 |
|--------|------|------|------|
| 股票代碼、名稱 | ✅ | ✅ | ✅ |
| 當前股價 | ✅ | ✅ | ✅ |
| K線歷史數據 | ✅ | ✅ | ✅ |
| P/E、P/B | ✅ | ❌ | △ |
| EPS、股利 | ✅ | ❌ | △ |
| 產業分類 | ✅ | △ | △ |
| 估值計算 | ✅ | ❌ | △ |

**說明**：
- ✅ 完全可用
- △ 部分可用或不穩定
- ❌ 通常不可用

## 💡 解決方案評估

### 方案 A：使用付費數據源

**IEX Cloud / Alpha Vantage 等**
- ✅ 完整的財務數據
- ✅ 美股支持完善
- ❌ 需要付費或 API 配額限制
- ❌ 需要 API Key 管理

### 方案 B：多源數據融合

**結合多個免費/付費 API**
- ✅ 覆蓋更多數據源
- △ 實現複雜度高
- △ 維護成本高
- ❌ 某些源可能也有限制

### 方案 C：現狀保持（當前選擇）

**繼續使用 Yahoo Finance**
- ✅ 實現簡單、無成本
- ✅ K線數據完整
- ✅ 對台股支持完善
- ❌ 美股財務數據缺失

### 方案 D：增強顯示邏輯

**對美股的特殊處理**
```javascript
// 若美股無基本面數據，自動隱藏相關功能
if (!isTW && !m.pe && !m.eps) {
  // 隱藏「估值模型試算」
  // 顯示「提示：美股數據源限制」
  // 只展示「技術面分析」
}
```

## 📖 用戶指南

### 查詢美股的正確預期

**可以期待**：
```
✅ 看價格走勢（K線、MA、成交量）
✅ 查歷史走勢（日線數據）
✅ 判斷技術面（MA20/MA60、RSI 等）
✅ 對標準普爾 500、QQQ 等 ETF
```

**不能期待**：
```
❌ 看 P/E、EPS 估值數據
❌ 自動估值計算
❌ 買賣建議價位計算
❌ 基本面深度分析
```

### 補充數據的途徑

```
對於美股財務數據，建議參考：

4. 公司 IR 官網（Investor Relations）
```
