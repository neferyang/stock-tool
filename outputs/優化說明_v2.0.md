### 2. **分級判斷策略（優先級遞降）**

```
L1（最高優先）：官方 API 資料 → Sector/Industry 中文匹配
    ↓
L2（高優先）：公司名稱高置信度關鍵字（如「台積電」→ IC設計）
    ↓
L3（中優先）：英文 Sector 判斷（美股）
    ↓
L4（低優先）：代號範圍推斷（台股 2000-6000 號段）
    ↓
L5（備選）：預設分類「未分類」
```

**範例判斷流程：**
- 輸入「2330」
- L1 檢查：TWSE API 資料 → 「半導體、積體電路」→ ✅ 判斷為 IC設計
- 置信度：**very-high**

### 4. **產業估值參數自動化**

每個細分產業關聯特定的估值參數：

```javascript
INDUSTRY_DB['ic-design'] = {
  name: 'IC設計',
  parent: '科技硬體',
  fairPE: 35,        // 合理本益比
  fairPB: 8,         // 合理股價淨值比
  fairDiv: 0,        // 合理現金股利
  fairYield: 0,      // 合理殖利率
  evEbitda: 20,      // 合理EV/EBITDA倍數
  description: '積體電路設計'
}
```

**使用效果：**
- 自動在估值模型中帶入產業特定參數
- 消除硬編碼邏輯，提高準確性和可維護性
- 新增產業時只需在 INDUSTRY_DB 中補充即可

---

### 5. **分類置信度指標**

每次分類都會返回置信度：

```javascript
{
  industry: 'ic-design',           // 產業代碼
  confidence: 'very-high',         // 置信度：very-high / high / medium / low
  details: {
    method: '公司名稱高置信',      // 判斷方法
    keyword: '台積電'              // 匹配的關鍵字
  }
}
```

**在 UI 中的呈現：**
- 高置信度（very-high / high）：直接顯示產業名稱
- 低置信度（medium / low）：加上 **ⓘ** 符號 + Tooltip 提示分類方法

## 📈 估值模型的聯動改進

### 自動適配產業估值參數

**之前（硬編碼）：**
```javascript
const fairPE = ind==='科技/半導體'?25:ind==='生技/醫療'?30:ind==='金融保險'?12:...
```
❌ 產業分類變更時容易出錯

**現在（參數化）：**
```javascript
const params = getIndustryParams(indKey);
const fairPE = params.fairPE || 16;  // 自動取得產業特定參數
```
✅ 產業與估值參數無縫銜接

## 🎓 估值模型覆蓋範圍

| 模型 | 適用情景 | 資料依賴 |
|------|--------|--------|
| **P/E 本益比評估** | 所有股票 | P/E 需 >0 且 <500 |
| **P/B 淨值估價** | 資本密集型（銀行、不動產） | P/B 需 >0 |
| **殖利率防線** | 配息股（電信、金融、高股息ETF） | 現金股利 >0 |
| **GGM 股息折現** | 穩定配息公司 | 現金股利 >0 |
| **DCF 簡化估值** | 有完整EPS的股票 | EPS 需 >0 |
| **EV/EBITDA** | 景氣循環股、高槓桿企業 | EV/EBITDA <200 |
| **RIM 剩餘盈餘模型** | 金融保險業 | ROE、P/B、淨值 |
| **PEG 本益成長比** | 高成長股（科技、生技） | PEG <20 |
