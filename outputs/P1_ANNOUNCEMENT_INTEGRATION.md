## 🔧 實施內容

### 1. 公告數據獲取

#### 方案 A：TWSE OpenAPI（優先）
```javascript
API 端點：
  https://openapi.twse.com.tw/v1/mopsfin/mops_t187ap05_${code}

優點：
  ✓ 官方授權數據
  ✓ 準確性 100%
  ✓ 實時更新
  ✓ 格式標準化

響應格式：
  [
    {
      Date: "2026-06-04",
      Title: "公告標題",
      Type: "M" | "C",  // M=重大, C=常規
      URL: "公告連結"
    },
    ...
  ]
```

#### 方案 B：MOPS 爬蟲（備援）
```javascript
網址：
  https://mops.twse.com.tw/mops/web/t05_query?code=${code}

爬蟲邏輯：
  1. Fetch HTML 頁面
  2. 正則匹配 <tr>...</tr> 表格行
  3. 提取日期、公司、標題、類型欄位
  4. 返回結構化數據

適用場景：
  ✓ API 暫時故障時
  ✓ 備份數據來源
  ✓ 提升可用性
```

#### 方案 C：回退連結（最後手段）
```javascript
若 A、B 都失敗：
  └─ 返回固定連結至 MOPS
  └─ 用戶可手動查詢完整公告
  └─ 保證應用正常運作（不阻塞）
```

### 2. 公告分類邏輯

#### 重大訊息識別
```javascript
判斷規則（優先級）：
  1. type === 'M'（API 標記）
  2. title.includes('重大')（關鍵字匹配）
  3. title.includes('公開發行')（補充判斷）

統計計數：
  m.majorNewsCount   // 重大訊息數
  m.minorNewsCount   // 常規公告數
  m.newsCount        // 總公告數
```

#### 數據結構
```javascript
m.news = [
  {
    date: "2026-06-04",
    title: "公告標題",
    type: "M" | "C",
    url: "連結" | null
  },
  ...
]

m.newsSource = "TWSE" | "MOPS" | "FALLBACK"
m.majorNewsCount = 3
m.minorNewsCount = 12
m.newsCount = 15
```

## 🔌 決策信號融合

### 買賣建議中的公告信號

#### 買進（Bull）場景
```
舊版建議：
  「🔴 估值偏低，可考慮買進」

新版建議（無重大訊息）：
  「🔴 估值偏低，可考慮買進」
  （保持不變）

新版建議（有重大訊息）：
  「🔴 估值偏低，可考慮買進 (🔴有重大訊息，建議謹慎)」
  + 警告文字：「但請先確認重大訊息內容」
```

#### 持有（Hold）場景
```
舊版建議：
  「🟡 估值合理，持有觀望」

新版建議（有重大訊息）：
  「🟡 估值合理，持有觀望 (🔴有重大訊息，建議謹慎)」
  + 警告文字：「本公司有 N 則重大訊息，建議查詢內容再做決策」
```

#### 賣出（Bear）場景
```
舊版建議：
  「🟢 估值偏高，建議減碼」

新版建議（有重大訊息）：
  「🟢 估值偏高，建議減碼」
  + 警告文字：「同時注意重大訊息，可能影響後續走勢」
```

### 公告摘要在建議中的位置
```
估值建議部分：
  1. 主標題（帶有重大訊息警示）
  2. 核心估值分析
  3. 技術面訊號
  4. 支撐/阻力位置
  5. 📰 最新公告 + 摘要     ← 【新增】
  6. ⚠️ 重大訊息警告        ← 【新增】
```

## 🔍 技術細節

### 錯誤處理與回退機制

```
場景 1：TWSE API 成功
  └─ 返回標準格式數據
  └─ m.newsSource = "TWSE"

場景 2：TWSE API 失敗，MOPS 爬蟲成功
  └─ HTML 解析後返回結構化數據
  └─ m.newsSource = "MOPS"

場景 3：都失敗，使用回退連結
  └─ 返回單條提示項目和連結
  └─ m.newsSource = "FALLBACK"

用戶影響：
  ✓ 任何情況下應用都能正常運作
  ✓ 無阻塞（8 秒超時後自動放棄）
  ✓ 降級優雅（回退方案仍有價值）
```

## 💡 設計思路

### 核心原則

1. **數據優先**
   - 優先使用官方 TWSE API
   - 備援 MOPS 爬蟲確保可用
   - 回退連結保證最後手段

2. **視覺優先級**
   - 重大訊息（紅色邊框）最醒目
   - 常規公告（灰色）次要位置
   - 決策建議中突出公告警告

3. **用戶決策輔助**
   - 在每個決策點顯示相關公告
   - 提供完整詳情的快速連結
   - 避免錯誤決策因素

4. **無感降級**
   - API 失敗自動回退
   - 不影響應用主功能
   - 用戶無需操作
