# 📱 財金早報強制更新功能 - 測試指南

## 🎯 功能概述

**強制更新按鈕** 讓用戶能手動刷新財金早報數據，無需等待每日 6:00 AM 的自動更新。

### 適用場景
- 🧪 測試自動更新機制是否正常工作
- 🔄 需要立即獲取最新市場數據
- 📊 驗證版本號和時間戳更新
- ⏰ 確認數據新鮮度

---

## 🚀 使用方法

### 步驟 1：進入財金早報分頁
1. 打開股票估價分析工具
2. 點擊右上角 **「📊 財金早報」** 分頁
3. 看到報表內容後，頂部會顯示：
   ```
   📊 2026年6月15日 市場風險指標儀表板  [🔄 強制更新]
   ```

### 步驟 2：點擊「🔄 強制更新」按鈕

按鈕顯示過程：
```
[🔄 強制更新]        ← 初始狀態（可點擊）
  ↓ (點擊)
[⏳ 更新中...]       ← 加載中（禁用）
  ↓ (1-2 秒)
[✅ 已更新]          ← 成功狀態（綠色）
  ↓ (2 秒後)
[🔄 強制更新]        ← 恢復原狀
```

**失敗時：**
```
[❌ 更新失敗]        ← 紅色失敗狀態
  ↓ (3 秒後)
[🔄 強制更新]        ← 恢復原狀
```

### 步驟 3：驗證更新效果

點擊按鈕後，檢查以下項目：

✓ **更新時間變化**
```
最後更新：2026/6/15 下午 6:15:00  ← 應該更新為點擊時間
🔄 手動強制更新 | ⏰ 每日 06:00
```

✓ **版本號變化**
```
v1.0.1  ← 版本可能遞增
2 版本歷史  ← 版本歷史記錄增加
```

✓ **更新來源顯示**
```
🔄 手動強制更新  ← 改為「手動強制更新」（而非自動）
```

## 🔬 測試用例

### 測試 1：基本功能驗證
**目標：** 確認按鈕能正常觸發更新

```
1. 記下初始更新時間
2. 點擊「🔄 強制更新」

4. 按鈕應顯示 「✅ 已更新」（綠色）
5. 頁面應重新加載最新數據
✅ 通過
```

### 測試 2：時間戳準確性
**目標：** 驗證時間戳正確更新

```
1. 記下當前時間（精確到秒）
2. 點擊「🔄 強制更新」
3. 等待更新完成（「✅ 已更新」）
4. 檢查「最後更新」時間
   期望：應接近按鈕點擊時間（差異 < 3秒）
✅ 通過
```

### 測試 3：版本歷史追蹤
**目標：** 驗證版本號和歷史記錄更新

```
1. 檢查初始版本號
   例：v1.0.0 (1 版本歷史)
2. 點擊「🔄 強制更新」
3. 檢查更新後版本號
   期望：v1.0.1 (2 版本歷史) ← 版本歷史記錄增加
✅ 通過
```

### 測試 4：多次更新
**目標：** 驗證多次更新無衝突

```
1. 點擊「🔄 強制更新」 → 確認成功
2. 等待 2 秒（按鈕恢復）
3. 再次點擊「🔄 強制更新」→ 確認成功
4. 重複 2-3 次
   期望：每次都應成功更新，無卡頓
✅ 通過
```

### 測試 5：網絡失敗處理（需斷網）
**目標：** 驗證錯誤提示正確

```
1. 斷網或關閉服務器
2. 點擊「🔄 強制更新」
3. 等待 1-2 秒
   期望：按鈕顯示「❌ 更新失敗」（紅色）
4. 檢查瀏覽器控制台 (F12)
   期望：應有錯誤消息：「強制更新失敗: ...」
5. 恢復網絡，重試
   期望：應成功
✅ 通過
```

## 📊 更新信息詳解

### 數據結構（daily-report.json）

```json
{
  "version": "1.0.1",           // 當前版本
  "versionHistory": [           // 版本歷史
    {"version": "1.0.0", "date": "2026-06-15 06:00:00", "status": "首次發佈"},
    {"version": "1.0.1", "date": "2026-06-15 06:15:00", "status": "數據更新"}
  ],
  "lastUpdated": "2026-06-15T06:15:00+08:00",        // 最後更新時間
  "lastForcedUpdate": "2026-06-15T06:15:00+08:00",   // 最後手動更新時間
  "autoUpdateSchedule": "每日 06:00 (台北時間)",      // 自動更新時間
  "updateSource": "手動強制更新",                    // 更新來源
  // ... 其他市場數據
}
```

### 更新來源標識

| 來源 | 說明 |
|------|------|
| `GitHub Actions 自動更新` | 每日 6:00 AM 由自動化工作流更新 |
| `手動強制更新` | 用戶手動點擊按鈕觸發 |

## ⚙️ 自動更新工作流

系統設置每日自動更新機制：

```
時間表：每日 06:00 台北時間 (UTC 22:00)
  ↓
GitHub Actions 自動執行
  ↓
運行 scripts/update-daily-report.js
  ↓
從 Yahoo Finance / TWSE 獲取最新市場數據
  ↓
更新 daily-report.json
  ↓
自動提交到 GitHub
  ↓
GitHub Pages 自動部署（1-2 分鐘內）
  ↓
網站顯示最新數據
```

**查看自動更新狀態：**
→ https://github.com/neferyang/stock-tool/actions

## 🐛 故障排除

### 問題 1：按鈕無反應
**症狀：** 點擊按鈕但沒有任何反應

**解決：**
1. 刷新頁面 (Ctrl+F5)
2. 檢查瀏覽器控制台 (F12)
3. 確認 `daily-report.json` 文件存在
4. 確認網絡連接正常

### 問題 2：顯示「❌ 更新失敗」
**症狀：** 強制更新失敗，按鈕顯示紅色

**檢查：**
1. 網絡是否連接正常
2. 服務器是否運行
3. `daily-report.json` 文件是否存在
4. 文件權限是否正確

**查看詳細錯誤：**
- 打開瀏覽器開發者工具 (F12)
- 查看 Console 標籤
- 查看錯誤消息

### 問題 3：時間不準確
**症狀：** 「最後更新」時間不是當前時間

**原因：**
- 服務器時區設置不對
- 使用了瀏覽器快取

**解決：**
1. 硬刷新頁面 (Ctrl+Shift+R)
2. 檢查系統時間是否正確
3. 確認時區設置為台北時間 (UTC+8)
