# 📊 Phase 2：數據正確性改善計畫

## ✅ Phase 2.1 完成 (數據驗證框架)

### 已實施內容
1. **異常檢測**
   - P/E > 500 或 < 0：標記為異常
   - P/B > 100：標記為異常（可能負淨值）
   - EPS ≤ 0：標記為虧損中，無法進行DCF

2. **數據一致性檢查**
   - PE 一致性：驗證 PE = Price / EPS
   - 偏差 > 10% 時警告（可能數據源不同）

3. **數據品質評分**
   - 統計有效數據個數（7項指標）
   - 評級：優（≥6個）/ 中（≥4個）/ 低
   - 顏色編碼：綠/黃/紅

### 測試驗證
- ✓ 異常檢測邏輯正確
- ✓ 一致性檢查有效
- ✓ 品質評分準確

## 🎯 Phase 2.2：基本面數據確認 (當前階段)

### 目標
驗證 P/E、P/B、EPS 的準確性，確保估值建議的可信度

### 數據來源優先級

#### 台股
```
優先級1：TWSE OpenAPI (台灣證券交易所)
  - 最權威的官方數據
  - P/E、P/B、DividendYield 準確度最高
  - 延遲：實時～1小時

優先級2：Yahoo Finance (美國)
  - 備用數據源
  - 對台股支持有限
  - 延遲：1～2小時

優先級3：計算值 (Price / PE → EPS)
  - 基於官方PE反推EPS
  - 最快但精度次於直接數據
```

#### 美股
```
優先級1：Yahoo Finance quoteSummary
  - 美股官方標準數據源
  - 準確度極高
  - 模塊：price, summaryDetail, defaultKeyStatistics

優先級2：Finnhub (需API Key)
  - 機構級別數據源
  - 更多財務指標
  - 當前已集成

優先級3：Alpha Vantage (免費備用)
  - 免費但速率有限
  - 可作為備援
```

### Phase 2.2 實施計畫

#### **步驟 1：標註數據來源**（優先度：高）
在每個估值指標旁添加數據來源標簽：
```
P/E: 15.2x (TWSE) ✓ 官方數據
P/B: 2.3x  (Yahoo) ⚠️ 非官方
EPS: 3.5元 (計算)  ℹ️ 基於TWSE PE反推
```

**實現方式：**
- 在go()函數中跟蹤每個數據的來源
- 在calcVals()中添加來源標簽
- UI展示來源信息

**預估工作量：** 30分鐘

---

#### **步驟 2：建立數據優先級選擇邏輯**（優先度：高）
當多個數據源都有時，自動選擇最可靠的：

```javascript
// 台股P/E優先級選擇
if (m.pe_twse) {
  m.pe = m.pe_twse;      // TWSE最優先
  m.peSource = 'TWSE';
} else if (m.pe_yahoo) {
  m.pe = m.pe_yahoo;      // Yahoo備用
  m.peSource = 'Yahoo';
} else if (m.pe && m.eps) {
  m.peSource = 'Calculated';  // 計算值
}
```

**實現方式：**
- 修改go()函數的數據整合邏輯
- 為每個指標添加來源追蹤變量
- calcVals()中輸出來源

**預估工作量：** 45分鐘
